# DocForge - Forge Your Requirements.

DocForge is a powerful, privacy-first Business Analysis tool that generates comprehensive requirements documents (BRD, FRD, PRD, CRD) directly in your browser.

> **Disclaimer:** Documents generated by DocForge are initial drafts designed to accelerate your workflow. They may contain inaccuracies or unresolved placeholders. Please review carefully, resolve any `[CLARIFICATION NEEDED]` tags, and apply your professional expertise to finalize the document. DocForge is an analyst co-pilot, not a replacement for human judgment and finesse.

## Prerequisites
- Node.js v18+

## Quickstart
1. Run the setup command to install dependencies and test browsers:
   ```bash
   npm run setup
   ```
2. Start the development server:
   ```bash
   npm run dev
   ```

## Running Tests
To run the Playwright End-to-End (E2E) tests:
```bash
npm run test:e2e
```

## Local LLM Setup Guide
For 100% privacy, you can use a local LLM instead of cloud providers.
1. Download and install [LM Studio](https://lmstudio.ai/) or [Ollama](https://ollama.ai/).
2. Load an instruction-tuned model (e.g. Llama 3 8B Instruct).
3. Set the context size to **8192** or higher to accommodate large document processing.
4. Start the local server. It typically runs on `http://localhost:1234/v1`.
5. In DocForge, open "Connection" settings, select "Local Server", and enter the URL.

## Processing Modes Explained
- **Semantic Injection (Section-by-Section):** Extracts and injects only relevant data per section using a local vector database. Best for large projects to prevent AI hallucination and token limit errors.
- **Comprehensive (Single File):** Sends the entire compiled source document at once. Best for small documents where deep cross-referencing is needed and fits within the model's context window.
